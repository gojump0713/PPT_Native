# DECISIONS — 결정·이유 기록

PRD 자율 실행 규칙에 따라, PRD·기능명세서에 명시되지 않았거나 해석이 필요한 사항의
결정 내용을 기록한다.

| # | 결정 | 이유 |
|---|---|---|
| D-01 | 전 스크립트를 클래식 `<script>`(IIFE)로 작성, ES 모듈 미사용 | `file://` 더블클릭 실행 시 모듈 스크립트는 CORS로 차단됨. "index.html을 열면 동작"이라는 완료 기준을 서버 없이도 충족하기 위함 |
| D-02 | Three.js는 `js/vendor/`에 로컬 포함하되 6개 씬은 Canvas 2D·CSS 3D·SVG로 구현 | 기능명세서의 6개 씬 요구 효과(회전목마=CSS 3D, 파티클≤200개=Canvas 2D, 다이어그램=SVG)에 WebGL이 필수적이지 않고, file:// 실행 호환성과 저사양 안정성이 우선. WebGL 감지는 저사양 판정에 활용 |
| D-03 | Lenis는 로컬 번들에 포함하되 슬라이드 이동은 자체 휠 누적기(임계값+쿨다운)로 처리 | 본 덱은 문서 스크롤이 없는 스텝 기반 구조로, 관성 스크롤보다 이동 오작동 방지(전환 잠금·연타 방지)가 중요. Lenis는 향후 롱스크롤 씬 추가 시 사용 가능 |
| D-04 | 슬라이드 마크업은 index.html에, 목차·노트·전환 메타데이터는 content.js에 분리 | 6장 각각이 고유 인터랙션 구조를 가져 완전 데이터 주도 템플릿화는 과설계. Content Driven 원칙은 메타데이터 계층에서 유지 |
| D-05 | S02 Before 이미지는 After 이미지를 참조로 한 img2img 변환으로 생성 | 명세 S02-AC-03(구도 어긋남 최소화) 충족을 위해 동일 카메라 구도 유지가 필수 |
| D-06 | 히어로 영상은 Kling 3.0 Turbo 10초 720p로 생성 | 배경용 풀블리드 + 어두운 오버레이가 얹혀 720p로도 시각 품질 충분. 필요 시 upscale로 교체 가능 (KNOWN_ISSUES 참조) |
| D-07 | 히어로 포스터는 After 이미지의 감광 변형본 사용 | 영상 첫 프레임 추출 도구(ffmpeg)가 환경에 없음. 포스터는 자동재생 차단 시의 대체 화면으로 동일 톤이면 충분 |
| D-08 | S03 회전목마는 카드별 `translateZ(-R) rotateY(θ) translateZ(R) rotateY(-θ)` 개별 배치 | 컨테이너 회전 방식으로는 텍스트 billboard(항상 정면) 요구를 만족하기 어려움. 선행 translateZ(-R)로 전면 카드가 1:1 원척도로 렌더되어 원근 과장 방지 |
| D-09 | 고정 스테이지 중앙정렬은 grid place-items가 아닌 절대배치+translate(-50%,-50%) | 스테이지가 뷰포트보다 클 때(스케일다운 상황) grid 중앙정렬은 트랙 오버플로로 실패함을 QA에서 확인 |
| D-10 | S06 프레임워크 점선 보드는 스텝 딤(35%) 대상에서 제외, 내부 모듈만 개별 딤 | 부모 컨테이너 opacity 딤은 자식 모듈의 하이라이트와 곱연산되어 명세 S06-SEQ-02의 "현재 영역 100% 밝기"를 깨뜨림 |
| D-11 | 도구 배지(PyTorch, Docker 등)는 텍스트 칩으로 구현 | 명세 S06-AST-01 지시 그대로 — 로고 이미지 상표 이슈 회피 |
| D-12 | 원본 4페이지 오탈자 '복학적인'→'복합적인' 교정 반영 | 기능명세서 9절 검수 비고 승계 (원작성자 최종 확인 필요) |
| D-13 | S05의 1:1 홍보 영상은 본편에 미삽입 | 기능명세서 S05-LAY-02 지시 그대로 — 16:9 스테이지와 비율 불일치 |
| D-14 | reduced-motion 시 GSAP 전역 타임스케일 3배 가속 + CSS 애니메이션 무력화 | COM-08 "0.2~0.3초 크로스페이드 대체"를 GSAP 기반 연출까지 일괄 적용하는 최소 침습 방식 |
| D-15 | 발표자 노트는 `window.open` + `document.write` 동일 출처 창으로 구현 | 오프라인·단일 배포 구조에서 별도 HTML 파일·서버 없이 동작. 팝업 차단 시 조용히 무시 |
| D-16 | 사용자 피드백에 따라 S01·S02칩·S04·S06을 자동 시퀀스로 전환 (01 명세의 스텝 분할 오버라이드) | 발주자 지시가 명세보다 우선. S06은 재지시로 2단계(트랙→자동 순차)로 재분할 |
| D-17 | 02덱 10장을 동일 index.html에 통합 (02 명세 1.1 승계) | 해시 네임스페이스 충돌 없음, Overview 4열 그리드로 확장 |
| D-18 | 배경 영상 4종은 Kling 3.0 Turbo 10초 720p로 생성, S08은 단일 영상의 낮(0.2~4.6s)/밤(5.4~9.7s) 구간을 currentTime 제어로 2상태 운용 | 명세 S08-AST-02 방식 준수. 16~20초 권장이나 생성 모델 최대 길이 제약으로 10초 내 2구간 구성 |
| D-19 | 영상 포스터는 헤드리스 브라우저 요소 스크린샷으로 추출 (ffmpeg 부재) | canvas.drawImage는 헤드리스에서 검은 프레임 반환 — 컴포지터 스크린샷 방식은 정상 |
| D-20 | S09 채팅 데모 질문/답변은 placeholder 스크립트로 구현, content 교체 가능 구조 | 02 명세 13절 — 실제 발표 맥락 스크립트는 별도 제공 예정, '연출된 예시' 캡션 병기 |
| D-21 | 전역 `[hidden] { display:none !important }` 추가 | display를 지정한 컴포넌트에서 hidden 속성이 무력화되는 표준 함정 방지 (QA에서 실증) |
| D-22 | 03덱 5장을 동일 index.html에 통합, scenes2의 `chapterScene`/`videoBg`를 `window.CHAPTER_SCENE`/`VIDEO_BG`로 노출해 03 챕터 타이틀에 재사용 | 챕터 타이틀 공통 패턴(CHP-01~05)을 덱 간 중복 구현 없이 승계. 해시 네임스페이스 충돌 없음, Overview 자동 확장 |
| D-23 | S02 IaC 영상을 3구간 개별 파일(`iac-seg-a/b/c.mp4`) 재생으로 구현 (명세의 단일 영상+타임코드 대안) | 구간별 파일은 구간 경계 정지(S02-AC-02)·역행 점프(S02-INT-01)가 currentTime 시크보다 프레임 정확. 단일 영상 필요 시 S08 방식으로 전환 가능(K-14) |
| D-24 | S02·S03·S05의 핵심 연출을 영상 없이도 완결되는 HTML 폴백으로 우선 구현 | 명세 S02-FBK-01의 '영상 제작 지연이 개발을 막지 않는다' 원칙. 저사양 모드와 미디어 미탑재 빌드 양쪽을 동일 코드로 커버 |
| D-25 | S05는 원본 4개 라벨을 리드문 기능 서술 근거로 허브+탭 목업 4종(강의자료실·학습/실습일지·과제제출·공지/QnA)으로 재구성 | 03 명세 6.1·8절 지시. '연출된 예시 화면' 캡션 병기(S05-CNT-07), 방향 확정 전 원작성자 검토 필요 |
